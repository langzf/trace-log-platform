# 容灾与备份方案

## 1. 可恢复目标
- RTO（恢复时间目标）：30 分钟
- RPO（恢复点目标）：5 分钟

## 2. 关键资产
- PostgreSQL 业务库
- Queue 元数据与 offset
- 配置中心数据（项目/执行器/策略）
- 审计日志与发布记录

## 3. 备份策略
- PostgreSQL：
  - 每日全量 + 每 5 分钟 WAL 增量
  - 保留 30 天
- 对象存储：版本化 + 生命周期策略
- 配置中心：每小时快照 + 变更审计

## 4. 容灾拓扑
- 主可用区 + 备可用区
- 控制面支持热备切换
- 执行器池跨机房分组

## 5. 故障切换流程
1. 触发条件满足（DB不可用 > 3min 或 P0链路不可用）
2. 宣布进入 DR 模式
3. 切换读写到备实例
4. 回放增量日志到最近一致点
5. 执行健康检查与抽样回归
6. 恢复服务并发布事件公告

## 6. 演练要求
- 每月一次 DR 演练
- 每季度一次全链路演练
- 演练结果形成复盘并追踪整改项
