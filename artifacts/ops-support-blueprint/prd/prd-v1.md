# PRD V1 - 运维支撑体系（内部）

## 1. 产品愿景
构建一个可持续进化的内部运维智能体体系，将问题处理流程从“人工串行”升级为“Agent 流水线 + 人工质量把关”。

## 2. 用户与角色
- 运营：关注反馈快速收敛与可视化跟踪。
- 研发工程师：关注根因证据与修复建议质量。
- SRE：关注稳定性、门禁、回滚和运行成本。
- 技术负责人：关注效率提升和长期 ROI。

## 3. 核心场景
1. 高频反馈自动收敛：自动分类、聚类、优先级分层。
2. 缺陷快速定位：联合日志/链路/代码上下文诊断。
3. 自动修复提 PR：低风险高置信任务自动推进。
4. 安全发布闭环：测试门禁 + 灰度观测 + 自动回滚。
5. 持续进化：失败复盘沉淀为技能和策略更新。

## 4. 功能范围

### Must
- 统一接入与标准化模型。
- 分类/聚类/优先级评分。
- 根因分析与置信度输出。
- 修复任务编排与执行器池。
- 自动 PR + 质量门禁。
- 可观测与审计。

### Should
- 灰度自动回滚。
- 复盘自动策略建议。
- 失败通知升级。

### Could
- 多项目跨仓联动修复。
- 自动变更摘要对比与风险解释。

### Won't (当前版本)
- 面向外部租户开放服务。
- 完整外部 IAM 与商业化计费。

## 5. 关键产品规则
1. 自动修复只到 PR，不直接生产发布。
2. 高风险任务必须人工审批。
3. 预算熔断优先级高于自动化完成率。
4. 任何失败都必须可复盘、可追责、可重放。

## 6. 非功能要求
- 关键链路可用性 >= 99.9%。
- P0/P1 问题 5 分钟内产出诊断结果。
- 任务状态更新延迟 < 10 秒。
- 任务执行日志保存 >= 180 天。

## 7. 里程碑
- M1（4周）：接入 -> 分类/聚类 -> 诊断闭环。
- M2（8周）：诊断 -> 修复 PR -> 门禁闭环。
- M3（12周）：发布回滚 -> 复盘进化闭环。
