# User Stories

| US ID | 角色 | 用户故事 | 优先级 | 关联需求 |
|---|---|---|---|---|
| US-001 | 运营 | 作为运营，我希望反馈自动分类，这样无需手工分拣 | Must | REQ-004 |
| US-002 | 运营 | 作为运营，我希望相似问题自动聚合，这样避免重复跟进 | Must | REQ-005 |
| US-003 | 研发 | 作为研发，我希望看到完整 trace 上下文，这样快速定位 | Must | REQ-002, REQ-007 |
| US-004 | 研发 | 作为研发，我希望系统输出根因证据链，这样可快速验证 | Must | REQ-007 |
| US-005 | 研发 | 作为研发，我希望自动生成修复 PR，这样减少重复编码 | Must | REQ-011 |
| US-006 | SRE | 作为 SRE，我希望任务具备幂等与重试，这样保障稳定性 | Must | REQ-003, REQ-009 |
| US-007 | SRE | 作为 SRE，我希望执行器并发隔离，这样避免任务冲突 | Must | REQ-010 |

| US-009 | Tech Lead | 作为技术负责人，我希望模型自动路由，这样平衡质量和成本 | Must | REQ-008, REQ-018 |
| US-010 | Tech Lead | 作为技术负责人，我希望可查看成本与效率看板 | Must | REQ-015 |
| US-011 | 审核人 | 作为审核人，我希望高风险任务必须人工审批 | Must | REQ-014 |
| US-012 | 发布负责人 | 作为发布负责人，我希望灰度异常自动回滚 | Should | REQ-013 |
| US-013 | 平台维护者 | 作为平台维护者，我希望有完整审计记录 | Must | REQ-016 |
| US-014 | 平台维护者 | 作为平台维护者，我希望失败可自动复盘并建议优化 | Should | REQ-017 |
| US-015 | 管理员 | 作为管理员，我希望统一管理仓库和执行器配置 | Must | REQ-020 |
