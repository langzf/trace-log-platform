# API Versioning Policy

## 1. 版本策略
- 主版本通过 URL 前缀控制：`/v1/...`。
- 向后兼容变更在小版本内完成，不破坏既有字段语义。

## 2. 兼容规则
- 允许新增可选字段。
- 禁止删除已发布字段。
- 禁止更改字段语义与类型。
- 枚举新增需保证客户端兜底。

## 3. 破坏性变更
- 需要升级到 `v2`。
- 提前至少一个发布周期提供双写/双读兼容层。
- 发布迁移指南与 deprecation 时间表。

## 4. 幂等与重试
- 写接口建议携带 `Idempotency-Key`。
- 服务端保留幂等窗口（默认 24 小时）。
- 重试需保持请求体一致。

## 5. 可观测要求
- 所有 API 需返回 `requestId`。
- 关键写操作需审计记录。
