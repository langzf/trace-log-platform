# 测试策略与质量门禁

## 1. 测试分层
1. 单元测试：分类规则、状态机迁移、路由策略
2. 集成测试：API + DB + Queue + Executor mock
3. 端到端测试：从事件接入到 PR 创建
4. 混沌测试：执行器失效、队列拥塞、模型高延迟

## 2. 门禁标准
- 单测通过率 100%
- 核心 E2E 用例通过率 >= 95%
- 关键模块变更覆盖率 >= 80%
- 无 P0/P1 未关闭缺陷

## 3. 回归策略
- 每日增量回归
- 每周全量回归
- 每次策略变更执行影子回归

## 4. 性能与容量测试
- 接入层 2,000 events/min 持续 60 分钟
- 并发修复任务 80 条
- 诊断 p95 保持 <= 5 分钟

## 5. 发布准入
- 通过质量门禁才允许进入灰度
- 灰度阶段关键指标恶化自动回滚
