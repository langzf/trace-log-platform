# ADR Log

## ADR-001：采用分层闭环架构而非单体流水线
- 状态：Accepted
- 决策：按“接入、理解、诊断、编排、执行、门禁、进化”分层。
- 备选：单体服务内部串行实现。
- 理由：降低耦合，便于扩展和故障隔离。

## ADR-002：模型分层路由（垂类主跑，通用兜底）
- 状态：Accepted
- 决策：分类/聚类等高频任务使用低成本模型；复杂诊断/修复使用高能力模型。
- 备选：全量使用统一强模型。
- 理由：在准确率可控前提下降低总体成本。

## ADR-003：自动修复止于 PR，不直接生产放行
- 状态：Accepted
- 决策：质量门禁与人工 CR 作为发布前强制环节。
- 备选：Agent 自动发布。
- 理由：当前阶段风险不可接受。

## ADR-004：任务驱动采用显式状态机
- 状态：Accepted
- 决策：RepairTask 使用可恢复状态机，支持幂等重试。
- 备选：隐式流程 + 日志回放。
- 理由：提升可观测性和稳定性。

## ADR-005：执行器池采用 worktree 隔离
- 状态：Accepted
- 决策：同仓多任务并发时每任务独立 worktree。
- 备选：共享工作目录。
- 理由：避免并发写冲突与污染。

## ADR-006：复盘结果驱动 Skill 与策略迭代
- 状态：Accepted
- 决策：失败任务必须生成 retro，进入优化流水线。
- 备选：人工不定期经验总结。
- 理由：形成可持续自进化闭环。
