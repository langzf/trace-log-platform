# 文档门禁检查清单

## 合并前必检
1. 是否在任务 PR 描述中引用了对应文档路径。
2. 文档是否包含“变更原因、影响范围、回退方案”。
3. API 变更是否同步 `api/openapi.yaml` 和 `error-code-spec.md`。
4. DB 变更是否包含 migration up/down 与回滚说明。
5. 关键任务是否包含测试结果与失败样例。
6. 运维相关改动是否补充告警和排障步骤。
7. `implementation/w1-w2/change-log.md` 是否追加记录。

## 评审质量要求
- 文档中不得出现“后补”或“TBD（非明确计划项）”。
- 关键参数需给默认值和可调范围。
- 所有缩写首次出现需展开说明。

## 拒绝合并条件
- 缺少任何强制文档。
- 文档与实现不一致。
- 无回滚方案。
- 无测试证据。
